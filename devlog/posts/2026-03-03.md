# A crackhead idea

At this point im pretty sure i'm just procrastiwriting about "whos flying this thing" part 2 - but instead of finishing that, I want to talk about a crackhead idea.

My line of thinking goes something like this. From my sample size n=1 experiment, ralph looping works. At least it works more than I was expecting it to work. This leads us to a question - what makes it work? Right now, my best hypothesis is feedback - correctly grounding the output in a "reality" which can be checked. Also maybe just not going outside the bounds of "learned" behaviour in the model, but thats hard to know about.

So anyway, since a nice simple example worked, I figured the most reasonable thing to do next is to try and do something insane. I was fishing around for something insane to try and build - when I had the idea to go close to home. We should try program a robot.

This is my hypothetical idea. I'm not sure yet if I can be bothered to actually try it, but i might. Maybe ill scale it back to trying the same thing in simulation so its less of a pain in the ass. The concept is simple right: we need 1. a way to program the robot, 2. a way to "run" the robot, and 3. a way to get feedback on what the robot did. Here's what im thinking:
- a small wheeled robot in a pen, controlled by a linux computer
- ssh access from a claude code machine, to the linux computer (or maybe just run the CC instance on the robot? - probably the limitation is we ideally want to be able to reboot the machine)
- a webcam set up with a view of the pen, connected to an mcp server, so claude can request an image of the current state of the pen
- _maybe_ an mcp server that lets it request help from me via a whatsapp message or something

I genuinely don't know how much of this would actually work / how much effort you would need to put into make this work. But I do know some of the pieces are there:
- ive seen claude successfully configure a nixos computer - all the way up to reloading the system
- i know it can interpret images
- i know it can write robotics / ROS code
- i know it can search the internet
- ive seen that it can perform work over some interesting horizion, using the right systems and structures
- ive even see that it can kinda _control_ robots using an mcp server: https://www.youtube.com/watch?v=EmpQQd7jRqs

so like, with the right structures in place, why not? Im not gonna ask too much - just like drive the robot to various waypoints in the pen or something.

Anyway, this is my crackhead idea - ill keep you updated if i get around it. Maybe ill video it and become a youtuber. Who knows
